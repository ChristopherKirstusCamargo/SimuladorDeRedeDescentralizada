# Decentralized Network Simulation Project

Hello! This is a simple decentralized network simulation project. Its purpose is to demonstrate, in a dynamic and visual way, how decentralized networks operate and how they behave under different conditions.

Since the simulator does not include extensive technical details, the following overview provides the essential concepts behind decentralized networks.

**Note:** This simulation is intended for educational purposes only. It does not replace in-depth networking studies. Always seek additional learning through reliable books, documentation, and technical resources.

## What is a Decentralized Network?

A decentralized network is a network architecture in which there is no single point of control. Unlike centralized networks, where one authority manages the entire infrastructure, decentralized networks distribute control and decision-making across multiple interconnected nodes.

Each node participates in the operation of the network, making the system more resilient and reducing its dependence on any single component.

## Why Use a Decentralized Network?

Decentralized networks offer several advantages. Below are some of the most significant benefits.

### Improved Security

Because control is distributed across multiple nodes, decentralized networks eliminate the single point of failure found in centralized architectures. If one node becomes unavailable or is compromised, the remaining nodes can continue operating, reducing the overall impact on the network.

Many decentralized systems also use end-to-end encryption or other cryptographic mechanisms to protect data during transmission, helping ensure confidentiality and integrity.

### Transparency

Another important characteristic of decentralized networks is transparency. In many decentralized systems, especially those based on blockchain technology, transactions and data changes are recorded in a shared, immutable ledger.

This allows network participants to verify and audit activities independently, reducing the risk of fraud, unauthorized modifications, or hidden changes.

> **Note:** Not every decentralized network uses a blockchain. Blockchain is one implementation of decentralization, not a requirement.

### Lower Latency

In some scenarios, decentralized networks can reduce communication latency by allowing data to travel directly between nodes instead of passing through a central server.

However, latency depends on many factors, including network topology, routing protocols, physical distance between nodes, and implementation. Therefore, decentralization does not always guarantee lower latency, although it can provide performance benefits in specific use cases.

## What Is the Difference Between Centralized, Decentralized, and Distributed Networks?

Although these architectures may appear similar, they differ in how control and resources are organized.

### Centralized Networks

In a centralized network, every device communicates through a central server or controller. This central component manages communication, authentication, and resource allocation.

While this model is simple to manage, it introduces a single point of failure. If the central server becomes unavailable, the entire network or its primary services may be affected.

### Decentralized Networks

A decentralized network consists of multiple interconnected nodes that share control. Instead of relying on a single authority, several nodes participate in managing the network.

This architecture improves fault tolerance and resilience because the failure of one node typically does not prevent the rest of the network from operating.

### Distributed Networks

A distributed network also operates without a central server, but its primary focus is on distributing computational resources and workloads among participating nodes.

Storage, processing, and data are shared across the network, allowing nodes to work together to complete tasks efficiently and increase scalability.

## Summary

The main difference between these three architectures lies in how control and resources are managed.

* **Centralized networks** rely on a single control point, making management simpler but introducing a single point of failure.
* **Decentralized networks** distribute control among multiple nodes, increasing resilience and fault tolerance.
* **Distributed networks** distribute both resources and workloads across participating nodes, enabling collaborative processing and improved scalability.

This simulation provides a simplified visualization of decentralized networking concepts to help users understand how these architectures function. It is intended as an introductory educational tool rather than a complete representation of real-world networking systems.
